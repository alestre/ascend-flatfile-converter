JSON json.org · RFC 8259
  • Objects: {"key": value} - keys must be double-quoted strings
  • Arrays: [value, value, ...]
  • Types: string (double quotes), number, true, false, null, object, array
  • No trailing commas after last element or key
  • No comments of any kind
Numbers must not have leading zeros (07 is invalid). Only double quotes for strings - single quotes are not allowed.
CSV RFC 4180
  • Each line is a record; fields separated by delimiter (default: ,)
  • Fields containing the delimiter, double quotes, or newlines must be quoted
  • A double quote inside a quoted field is escaped by doubling: ""
  • First row is conventionally (not required) the header row
  • No type system - all values are strings
No single universal standard exists. RFC 4180 specifies CRLF line endings, but LF-only is widely accepted. Trailing newlines are optional.
XML W3C XML
  • Must have exactly one root element
  • Tags are case-sensitive: <Tag> ≠ <tag>
  • All tags must be closed: <el/> or <el></el>
  • Attribute values must be quoted: <el attr="value">
  • Special chars: &amp; &lt; &gt; &quot; &apos;
Well-formed (syntactically correct) ≠ valid (conforms to a schema). Comments cannot contain --. Processing instruction: <?xml version="1.0" encoding="UTF-8"?>
YAML YAML 1.2
  • Indentation-sensitive: spaces only (no tabs), child indented more than parent
  • Key-value pairs: key: value
  • List items: - item
  • Strings usually unquoted; quote if value contains : # { [ etc.
  • Reserved scalars: true, false, null, ~
YAML 1.1 treats yes/no/on/off as booleans; YAML 1.2 does not. A colon must be followed by a space to be treated as a key separator.
TOML TOML v1.0
  • Key-value pairs: key = value
  • Sections (tables): [table]
  • Arrays of tables: [[array]]
  • String types: "basic", 'literal', """multi-line"""
  • Datetime: 1979-05-27T07:32:00Z
Keys cannot be redefined once set. Bare keys: only A-Za-z0-9_-. A table header must appear before any of its sub-key assignments.
